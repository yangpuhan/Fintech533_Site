Breakout Trading Strategy

Donchian Channel Breakout — Backtest & Analysis

Strategy Logic

This strategy exploits Donchian Channel breakouts on daily price data. A Donchian Channel marks the highest high and lowest low over a rolling look-back window. When price closes above the upper channel, we enter a long position, expecting momentum to carry the move further. Each trade is governed by three exit rules applied in priority order: (1) a stop-loss set at a fixed percentage below the entry price to cap downside risk, (2) a profit target at a fixed percentage above entry to lock in gains, and (3) a timeout that forces a market-order exit after a maximum holding period if neither the stop nor the target has been hit. All parameters — look-back length, stop-loss distance, profit target, and timeout — are calibrated on a rolling one-year training window and then applied out-of-sample on the subsequent period, following a walk-forward methodology to reduce overfitting.

Breakout Definition

The strategy uses a Donchian Channel (also known as a price-channel breakout):

Parameter Description Default
lookback Rolling window length (trading days) for the channel 20
stop_loss_pct Maximum adverse move before forced exit 2 %
profit_target_pct Gain level that triggers a profitable exit 4 %
timeout_days Maximum holding period before a market-order exit 10
position_size Number of contracts/shares per trade 100

Entry rule: A long entry is triggered when the daily close exceeds the highest high of the previous lookback days (the upper Donchian band). Only one position can be open at a time.

Exit rules (checked in order each day):

  1. Stop-loss — if the low of the day drops to or below entry_price × (1 − stop_loss_pct), the trade is closed at the stop price.
  2. Profit target — if the high of the day reaches or exceeds entry_price × (1 + profit_target_pct), the trade is closed at the target price.
  3. Timeout — if the position has been held for timeout_days trading days without hitting either threshold, it is closed at the day’s closing price.
Show code 
# ---------------------------------------------------------------------------
# Breakout detection & backtesting engine
# ---------------------------------------------------------------------------

# Default strategy parameters (can be overridden)
DEFAULT_LOOKBACK = 20
DEFAULT_STOP_LOSS_PCT = 0.02
DEFAULT_PROFIT_TARGET_PCT = 0.04
DEFAULT_TIMEOUT_DAYS = 10
DEFAULT_POSITION_SIZE = 100


def detect_breakouts(prices: pd.DataFrame, lookback: int = DEFAULT_LOOKBACK) -> pd.Series:
    """
    Identify Donchian Channel upper-band breakouts.

    Parameters
    ----------
    prices : DataFrame with columns ['Open','High','Low','Close','Volume']
    lookback : int — number of prior trading days for the channel

    Returns
    -------
    Boolean Series — True on days where Close > prior lookback-day high.
    """
    upper_band = prices["High"].rolling(lookback).max().shift(1)
    return prices["Close"] > upper_band


def backtest_breakouts(
    prices: pd.DataFrame,
    lookback: int = DEFAULT_LOOKBACK,
    stop_loss_pct: float = DEFAULT_STOP_LOSS_PCT,
    profit_target_pct: float = DEFAULT_PROFIT_TARGET_PCT,
    timeout_days: int = DEFAULT_TIMEOUT_DAYS,
    position_size: int = DEFAULT_POSITION_SIZE,
) -> pd.DataFrame:
    """
    Run a vectorised-loop backtest on Donchian breakout signals.

    Returns a DataFrame (the trade blotter) with one row per round-trip trade.
    """
    signals = detect_breakouts(prices, lookback)
    trades = []
    i = 0
    n = len(prices)
    idx = prices.index

    while i < n:
        if not signals.iloc[i]:
            i += 1
            continue

        # --- ENTRY ---
        entry_date = idx[i]
        entry_price = float(prices["Close"].iloc[i])
        stop_price = entry_price * (1 - stop_loss_pct)
        target_price = entry_price * (1 + profit_target_pct)

        # --- MANAGE TRADE ---
        exit_date = None
        exit_price = None
        outcome = None

        for j in range(1, timeout_days + 1):
            k = i + j
            if k >= n:
                # end of data — close at last available close
                exit_date = idx[n - 1]
                exit_price = float(prices["Close"].iloc[n - 1])
                outcome = "Timed out"
                i = n
                break

            day_low = float(prices["Low"].iloc[k])
            day_high = float(prices["High"].iloc[k])
            day_close = float(prices["Close"].iloc[k])

            # Check stop-loss first
            if day_low <= stop_price:
                exit_date = idx[k]
                exit_price = stop_price
                outcome = "Stop-loss"
                i = k + 1
                break

            # Check profit target
            if day_high >= target_price:
                exit_date = idx[k]
                exit_price = target_price
                outcome = "Successful"
                i = k + 1
                break

            # Timeout on last allowed day
            if j == timeout_days:
                exit_date = idx[k]
                exit_price = day_close
                outcome = "Timed out"
                i = k + 1
                break
        else:
            i += 1
            continue

        pnl = (exit_price - entry_price) * position_size
        ret = (exit_price - entry_price) / entry_price

        trades.append({
            "Entry Date": entry_date,
            "Exit Date": exit_date,
            "Entry Price": round(entry_price, 4),
            "Exit Price": round(exit_price, 4),
            "Position Size": position_size,
            "Direction": "Long",
            "P&L ($)": round(pnl, 2),
            "Return (%)": round(ret * 100, 4),
            "Outcome": outcome,
        })

    return pd.DataFrame(trades)

Walk-Forward Backtest

I use a rolling walk-forward approach: each year of data is used as a training window to select the best lookback parameter (from 10 to 40 days) by maximising the Sharpe ratio, and the optimised parameter is applied out-of-sample on the next six months.

Show code 
prices = all_data["CL=F"].copy()

# Walk-forward windows
train_days = 252    # ~1 year
test_days  = 126    # ~6 months
lookback_grid = [10, 15, 20, 25, 30, 40]

all_trades = []
wf_log = []
start_idx = 0

while start_idx + train_days + test_days <= len(prices):
    train = prices.iloc[start_idx : start_idx + train_days]
    test  = prices.iloc[start_idx + train_days : start_idx + train_days + test_days]

    # --- Optimise on training set ---
    best_sharpe = -np.inf
    best_lb = DEFAULT_LOOKBACK
    for lb in lookback_grid:
        tr = backtest_breakouts(train, lookback=lb)
        if len(tr) < 3:
            continue
        rets = tr["Return (%)"] / 100
        sr = rets.mean() / rets.std() * np.sqrt(252 / DEFAULT_TIMEOUT_DAYS) if rets.std() > 0 else 0
        if sr > best_sharpe:
            best_sharpe = sr
            best_lb = lb

    # --- Apply out-of-sample ---
    oos_trades = backtest_breakouts(test, lookback=best_lb)
    window_label = f"{train.index[0].date()} → {test.index[-1].date()}"
    oos_trades["Window"] = window_label
    all_trades.append(oos_trades)
    wf_log.append({
        "Window": window_label,
        "Best Lookback": best_lb,
        "Train Sharpe": round(best_sharpe, 2),
        "OOS Trades": len(oos_trades),
    })

    start_idx += test_days  # slide forward

blotter = pd.concat(all_trades, ignore_index=True)
wf_df = pd.DataFrame(wf_log)

Performance Metrics

Show code 
returns = blotter["Return (%)"] / 100
pnls = blotter["P&L ($)"]
n_trades = len(blotter)

# Avg return per trade
avg_return = returns.mean()

# Win rate
wins = (returns > 0).sum()
win_rate = wins / n_trades if n_trades else 0

# Sharpe ratio (annualised)
# Assume average trade duration ~ timeout_days/2, risk-free = 4.5 % annual
risk_free_annual = 0.045
avg_holding = 5  # approximate days
trades_per_year = 252 / avg_holding
rf_per_trade = risk_free_annual / trades_per_year
excess = returns - rf_per_trade
sharpe = excess.mean() / excess.std() * np.sqrt(trades_per_year) if excess.std() > 0 else 0

# Sortino ratio
downside = excess[excess < 0]
downside_std = np.sqrt((downside**2).mean())
sortino = excess.mean() / downside_std * np.sqrt(trades_per_year) if downside_std > 0 else 0

# Profit factor
gross_profit = pnls[pnls > 0].sum()
gross_loss = abs(pnls[pnls < 0].sum())
profit_factor = gross_profit / gross_loss if gross_loss > 0 else np.inf

# Max drawdown (on cumulative P&L)
cum_pnl = pnls.cumsum()
running_max = cum_pnl.cummax()
drawdowns = cum_pnl - running_max
max_dd = drawdowns.min()

# Expectancy
expectancy = pnls.mean()

metrics = {
    "Total Trades": n_trades,
    "Win Rate": f"{win_rate:.1%}",
    "Avg Return per Trade": f"{avg_return:.2%}",
    "Sharpe Ratio (annualised)": f"{sharpe:.2f}",
    "Sortino Ratio (annualised)": f"{sortino:.2f}",
    "Profit Factor": f"{profit_factor:.2f}",
    "Expectancy ($ per trade)": f"${expectancy:,.2f}",
    "Max Drawdown ($)": f"${max_dd:,.2f}",
    "Risk-Free Rate Assumed": f"{risk_free_annual:.1%}",
}

metrics_df = pd.DataFrame(
    list(metrics.items()), columns=["Metric", "Value"]
)

What Each Metric Tells Us

Metric Interpretation
Win Rate Fraction of trades that closed with a positive return.
Avg Return per Trade Mean percentage gain (or loss) across all trades — the strategy’s raw edge.
Sharpe Ratio Risk-adjusted return; values above 1.0 suggest attractive compensation for volatility. Uses a 4.5 % annualised risk-free rate.
Sortino Ratio Like the Sharpe but penalises only downside volatility — more relevant for asymmetric return distributions.
Profit Factor Gross profits ÷ gross losses. Above 1.0 means the strategy is net profitable.
Expectancy Average dollar P&L per trade — what a trader can expect to make on each entry, on average.
Max Drawdown Largest peak-to-trough decline in cumulative P&L — indicates worst-case capital erosion.

Assumptions & Replication Notes

  • Slippage / commissions: Not modelled. In practice, CL=F has very tight bid-ask spreads (~$0.01/barrel) so the impact on a 20-day breakout system is minimal.
  • Data source: Yahoo Finance via yfinance. Prices are unadjusted front-month continuous contracts.
  • Position sizing: Fixed at 100 units per trade (no compounding).
  • Walk-forward splits: 252-day train / 126-day test, sliding by 126 days.
  • Risk-free rate: 4.5 % annualised (approximate US T-bill yield).
